# 05. 注意すべき貿易実務ポイント

アプリ設計に直接影響する順に記載する。

## 1. PI・CI・PL の三点整合性(最重要)

税関・銀行・バイヤーは書類間の食い違いを見る。**数量・金額・重量・荷受人・インコタームズが書類間で矛盾すると、通関遅延・受取拒否・送金トラブルの原因**になる。

- → 本アプリでは案件(deals)を単一の真実源とし、PI/CI/PLはすべて案件から生成する。
- → PLにはCI番号を参照記載し、突合を容易にする。
- → 発行後に案件を変えた場合、旧書類はVoid+改訂版発行(Rev管理)で履歴を残す。

## 2. 発行済み書類の不変性

インボイスは商取引・通関の証憑であり、**発行後にサイレントに書き換わってはならない**。

- → `documents.data` スナップショット方式(§02)。マスタ編集が過去書類に波及しない。
- → 修正は必ず「Void → Rev.N 再発行」。どの版をバイヤーに送ったか履歴で追える。

## 3. HSコード(抹茶)

緑茶のHSコードは **0902.10(内容量3kg以下の小売包装)/ 0902.20(3kg超)**。100gパウチ販売なら0902.10系だが、**下6桁以降の細分と輸入国側の分類は国ごとに異なる**。

- → HSコードは商品マスタで管理しつつ、**案件・書類ごとに上書き可能**にする(仕向国によって適切な細分が変わるため)。
- → 最終確定は通関業者・フォワーダーに確認する運用とし、アプリはその値を記録する役割に徹する。

## 4. インコタームズは「規則+場所」で記録する

"FOB" 単体では不完全。**"FOB Hakata", "DAP London", "EXW Yame Warehouse" のように場所込み**で初めて意味を持つ(Incoterms 2020)。

- → DBは自由入力1カラム(`incoterms`)とし、入力UIで「規則(選択)+場所(入力)」を結合して保存する。
- → 送料をPIに含めるか(CIF/CPT/DAP)、含めないか(EXW/FOB)は条件次第。送料行の表示有無を発行フォームで制御できるようにする。

## 5. 100%前払い条件とステータスのガード

この事業の生命線は「**入金確認前に生産・出荷しない**」こと。

- → ステータス遷移にソフトガードを入れる: 入金記録(payments)がない案件を `Paid` 以降に進めようとしたら警告を出す(ブロックはしない。実務の例外に対応するため)。
- → 入金証憑(payment_proof)のアップロードを `Paid` 遷移時に促す。
- → Wise等の着金額は銀行手数料で端数が欠けることがある。PIに手数料負担条件(例: "All bank charges are to be borne by the buyer / OUR")を明記できるNotes欄を用意。

## 6. 食品輸出ならではの規制(COA・LOTの意義)

抹茶は食品であり、仕向国ごとに要求が異なる。**COAとLOTトレーサビリティは「あれば便利」ではなく通関・取引の必須要件**になり得る。

- **EU**: 茶葉の残留農薬MRLが極めて厳格。バイヤーからLOTごとのCOA(農薬・微生物・重金属)を求められるのが通例。輸入者はEORI必須。
- **米国**: FDA食品施設登録(自社or製造元)と、出荷ごとの **Prior Notice** が必要。バイヤー側はFSVP対応でCOA・製造者情報を求めてくる。
- **放射性物質検査**: 福島事故由来の検査・証明要求は大半の国で撤廃済みだが、一部の国・バイヤーは自主的に証明を求めることがある。
- → LOTをproductsと別テーブルにし、**COAをLOT単位で添付**できる設計(§02)はこのため。案件明細にLOTを紐付ければ「どのバイヤーにどのLOTを出したか」が追跡でき、万一のリコール時に対応できる。

## 7. 日EU EPA等の関税優遇(原産地申告)

日本産抹茶はEU向けで**日EU EPAにより関税撤廃**の恩恵を受けられるが、それには**インボイス上の原産地申告文(Statement on Origin)**が必要(6,000ユーロ超はREX相当の輸出者登録番号が必要)。日英EPAも同様の仕組み。

- → MVPではCIの備考に手書きできれば十分。将来機能として定型申告文の自動挿入をロードマップに含めた(§04)。
- → バイヤーの関税コスト削減は営業上の強い武器になるので、早めの対応価値が高い。

## 8. インボイス金額の正確性(アンダーバリュー禁止)

CIの金額は関税・輸入VATの課税標準。バイヤーから「低く書いてほしい」と依頼されても**アンダーバリューは輸入国での脱税・押収リスク**であり応じない。Reason for export: **Sale** を明記し、無償サンプルは別途 "Sample, no commercial value"(ただし実勢価格記載が原則)の書式で扱う。

- → 書類種別として将来「Sample Invoice」を追加する余地を残す。

## 9. 重量・梱包情報の正確性

Net/Gross weightと寸法は**運賃計算(容積重量)と通関検査の基準**。実測とPLの乖離は検査・追加請求の元。

- → deal_cartonsで箱単位に記録し合計を自動計算(手計算ミス排除)。容積重量(L×W×H/5000)の参考表示も入れると運賃見積に便利。

## 10. VAT / EORI の管理

EU/UKバイヤーとの取引ではVAT番号・EORI番号がCIに載っていないと通関が止まることがある。

- → 顧客マスタに保持し(§02)、CI生成時に**EU/UK仕向けでEORIが空なら警告**を出す。

## 11. 通貨・見積有効期限

為替変動があるため、外貨建てPIには**有効期限(Validity: 30 days等)を必ず記載**する。期限切れ後の入金には再発行で対応。

- → PI発行フォームにValidity入力欄(デフォルト30日)を持たせ、Notesに自動挿入。

## 12. プライベートラベル(OEM)の表示責任

輸入国の食品表示規制(EU FIC、米国FDAラベル規則等)への適合は**原則として輸入者(バイヤー)の責任**だが、ラベル貼付を自社加工で行う以上、トラブル時に巻き込まれる。

- → 案件ファイルの `product_label` カテゴリで**バイヤー承認済みラベルデータを必ず保管**し、「承認済みラベルを貼付した」証跡を残す運用にする。QC写真(qc_photo)も同様。

## 13. 品質保持とリードタイム

抹茶は光・酸素・熱で急速に劣化する。窒素充填・脱酸素剤・アルミパウチは説明文だけでなく品質クレーム予防の実務。**製造日・賞味期限(best_before)をLOTに記録**し、出荷時に残存賞味期限をバイヤー要求(例: 残12ヶ月以上)と照合できるようにする。
