# 抹茶輸出販売管理アプリ — 要件定義・設計ドキュメント

対象事業: **MATCHA NINJA / WAGYUNINJA** — 日本産抹茶の海外卸・OEM・プライベートラベル輸出

本アプリは会計ソフトの置き換えではなく、**海外顧客との商談〜見積〜PI〜入金〜加工〜発送〜完了までを一元管理する社内業務ツール**である。

## 技術スタック(結論)

提案いただいた構成をそのまま推奨する。これが現状もっとも簡単な構成でもある。

| レイヤ | 採用技術 | 理由 |
|---|---|---|
| フレームワーク | **Next.js (App Router / TypeScript)** | 画面・API・PDF生成を1リポジトリで完結。Vercelに無料枠でデプロイ可 |
| DB | **Supabase (PostgreSQL)** | 認証・DB・ファイルストレージが1サービスで揃う。RLSで将来の権限分離にも対応 |
| 認証 | **Supabase Auth** (メール+パスワード) | MVPは管理者1名。招待制でスタッフ追加可能 |
| ファイル保存 | **Supabase Storage** | 案件ごとのファイル保管。Google Drive連携は将来フェーズ(§04参照) |
| PDF生成 | **@react-pdf/renderer** | Reactコンポーネントとして帳票を定義。Puppeteer不要でVercelのサーバーレス環境で安定動作 |
| UI | **Tailwind CSS + shadcn/ui** | 社内ツールに十分な品質のUIを最速で構築 |
| ホスティング | **Vercel** | Next.jsと相性最良。社内数名なら無料〜低コスト |

### 「もっと簡単な構成」を検討した結果

- **Airtable / Notion**: 顧客・案件管理はできるが、**PI/CI/PLの精密なPDF生成と自動採番・書類スナップショット保存ができない**ため不採用。貿易書類は体裁とデータ整合性が命であり、ここを外部ツールで妥協すべきでない。
- **Google スプレッドシート + GAS**: 初期コストは低いが、ステータス管理・ファイル管理・権限で早晩破綻する。
- 結論: **Next.js + Supabase が「簡単さ」と「業務要件」の交点として最適**。運用コストは月0〜数千円。

## 設計上の最重要原則

1. **発行済み書類は不変(イミュータブル)**: PI/CI/PLは発行時点のデータをJSONスナップショットとして保存し、後から顧客・商品・案件を編集しても発行済みPDFの内容は変わらない。修正は改訂版(Rev)の再発行で行う。
2. **社内原価は絶対にPDFに出さない**: 加工費の社内原価メモは専用テーブルに分離し、PDF生成処理のデータフローに一切含めない(§02・§03参照)。
3. **PI・CI・PLの数値整合性**: 同一案件から生成される書類間で金額・数量・重量が矛盾しないよう、案件データを単一の真実源(Single Source of Truth)とする。
