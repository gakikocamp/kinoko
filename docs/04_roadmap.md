# 04. 開発ステップ・最初に作る最小機能・将来機能

## 1. 最初に作るべき最小機能(Milestone 1 = 業務で使い始められる最小線)

**「PIを発行して前払いを受け取る」ところまで**を最小スコープとする。理由: この事業は100%前払いが取引条件であり、**PIが出せれば商売のボトルネックが解消する**。CI/PLは出荷時まで不要なので次のマイルストーンでよい。

Milestone 1 に含むもの:

1. ログイン+MFA強制(管理者1名) — F1-1〜F1-3
2. 設定画面(自社情報・銀行/Wise・署名/ロゴ) — F10-1〜F10-3
3. 顧客CRUD+自動採番 — F2-1〜F2-3
4. 商品CRUD(LOT・COAは後回し可) — F3-1, F3-2
5. **国マスタ(シード投入+可否バッジ+ブロック)** — F8-1〜F8-3(調べていない国に売れない構造はMVPの安全装置なので初期から入れる)
6. 案件CRUD+ステータス+金額自動計算+加工費(顧客向け) — F4-1〜F4-6, F5-1
7. **PI発行**(採番・スナップショット・PDF保存・再ダウンロード) — F6-1, F6-4〜F6-6
8. 監査ログ基盤(主要操作の記録のみ。閲覧画面は後回し可) — F9-1

含まないもの: CI/PL、ファイル管理全般、入金記録、LOT/COA、社内原価メモ、要件チェックリスト(F8-4) → Milestone 2以降。

## 2. 開発ステップ

| Phase | 内容 | 目安 |
|---|---|---|
| **0. 基盤** | Next.js(App Router/TS)+Tailwind+shadcn/ui 雛形、Supabaseプロジェクト作成、Auth+MFA+middleware、DBマイグレーション一式(§02のDDL全部+国マスタシード投入)、監査ログ基盤、Cloudflare Workersデプロイパイプライン(@opennextjs/cloudflare+GitHub Actions — §09) | 3〜4日 |
| **1. マスタ** | 設定画面 / 顧客CRUD / 商品CRUD / 国マスタ画面(可否バッジ・編集)、`next_doc_number()` による採番、一覧・検索 | 4〜6日 |
| **2. 案件** | 案件CRUD、ステータス遷移+履歴、明細・金額計算、加工費(顧客向け+社内原価タブ)、国別ブロック/警告 | 4〜6日 |
| **3. PI発行** | 発行フォーム(案件から初期値展開→編集)、`@react-pdf/renderer` テンプレート、プレビュー、発行確定(採番+スナップショット+Storage保存)、書類一覧 | 4〜6日 |
| ― | **← ここで Milestone 1 リリース。実案件で使い始める** | 計 約2〜3週間 |
| **4. 出荷書類** | CI発行(発送方法・追跡番号)、PL発行(カートン明細入力・重量集計)、Void/改訂版発行 | 4〜6日 |
| **5. ファイル・入金** | 案件ファイル管理(カテゴリ別アップロード)、LOT/COA管理、入金記録+証憑、ダッシュボード | 4〜6日 |
| **6. 仕上げ** | 加工費テンプレート、ステータス操作のガード(§05-5)、監査的な微修正、実運用フィードバック反映 | 継続 |

進め方の注意:

- **DDLはPhase 0で全テーブル作成しておく**(後からのスキーマ変更コストを避ける)。画面は段階的に作る。
- Phase 3の完了条件は「実際のバイヤーに送れる品質のPI PDFが出ること」。帳票の体裁確認に時間を割く。
- 各Phaseの終わりに実データ(実在の商品・顧客)を入れて業務担当者がレビューする。

## 3. 将来的に追加すべき機能(優先度順)

| 優先 | 機能 | 内容 |
|---|---|---|
| 高 | **Quotation PDF** | PI以前の見積書。PIとほぼ同テンプレートで安価に追加できる |
| 高 | **Google Drive連携** | 発行PDF・アップロードファイルを案件フォルダ(`/Deals/DEAL-2026-0001/`)へ自動同期。Drive APIのサービスアカウントで実装。Storageを真実源としDriveは複製 |
| 高 | **スタッフ権限分離** | role別RLS: viewerは閲覧のみ、staffは原価(`cost_price`・`deal_processing_costs`)非表示 |
| 高 | **メール送信** | PI/CI/PLを画面からバイヤーへ直接送信(Resend等)。送信履歴を案件に記録 |
| 中 | **原産地申告文の自動挿入** | 日EU EPA・日英EPA向けStatement on OriginをCIにオプション追記(§05-7) |
| 中 | **在庫・LOT引当** | LOTごとの在庫数量を管理し、案件出荷で引き落とす |
| 中 | **為替レート補助** | JPY建て原価×為替レートで外貨見積の粗利をリアルタイム表示 |
| 中 | **ダッシュボード強化** | 月次売上、国別・顧客別売上、パイプライン金額、粗利(社内のみ) |
| 中 | **リマインダー** | PI発行後N日未入金、出荷予定日超過、国マスタの確認日1年超過などをメール/Slack通知 |
| 中 | **制裁リストスクリーニングAPI連携** | 顧客登録時にOFAC SDN・EU制裁リスト等と自動照合(MVPは手動確認チェック — §07-5) |
| 中 | **GDPR対応の顧客匿名化** | EU顧客からの削除要請に応じ個人名・連絡先をマスクする処理(§08-4) |
| 中 | **サンプル管理** | サンプル発送の追跡・フォローアップ管理(Sample Sentステータスの中身) |
| 低 | **会計ソフト連携** | freee / マネーフォワードへのCSVエクスポート(会計の置き換えはしない方針を維持) |
| 低 | **顧客ポータル** | バイヤーが自分の注文状況・書類・追跡番号を見られる外部ページ |
| 低 | **監査ログ** | 全レコードの変更履歴(誰が・いつ・何を) |
| 低 | **多通貨・複数銀行口座** | 通貨別の受取口座を自動で書類に出し分け |
