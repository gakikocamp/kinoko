# 制作過程・意思決定の記録

アネラカフェ福岡サイトをゼロから作るにあたっての検討と決定の記録です。
「なぜこうなっているか」を引き継ぎ先（人／AIエディタ）が把握するための背景資料です。

## 出発点
- リポジトリ `gakikocamp/kinoko` はほぼ空（キノコキャンプのロゴ画像2点のみ、2019年以降放置）。
- 要件: ①カフェ紹介 ②里親募集 ③B型就労支援の利用者募集 ④スタッフ求人。
- 運用方針: 手間を増やさない。お知らせは note 更新→RSS。わんこ一覧は作らず Instagram へ。

## ヒアリングで確定した方針
| 論点 | 決定 |
|------|------|
| サイト構成 | **店舗ごとに2サイト**（別ドメイン想定） |
| note | **全店共通で1アカウント**（同じRSSを両サイトに表示） |
| 応募・問い合わせ | **フォームは作らない**。メール／電話／Instagram リンクで受ける |
| ホスティング | お任せ → 当初 GitHub Pages で仮確認、最終は **Cloudflare Pages** |
| メニュー | **基本ドリンクバー**のためメニューページは廃止。トップに「ご利用案内」 |
| 画像 | Instagram は自動取得不可（要ログイン）。**公式埋め込み（無料）**＋**手動で画像配置**の二段構え |
| ブランド | 「Anela＝天使（ハワイ語）」を起点に Teal×Gold×Coral×Ivory のカラー設計 |

## 構成の要点（なぜモノレポか）
「2サイトに分ける」一方で note・里親・B型・求人・デザインは共通。二重管理を避けるため
**pnpm モノレポ**にし、`packages/shared` に共通部品（レイアウト／ページ本体／コンテンツ／note取得）を集約。
各サイトは店舗固有データ（`sites/<store>/src/data/store.ts`）だけを持つ薄いラッパー。
→ デザインや文面の修正は1か所で両店に反映される。

## 主な実装の経緯（時系列の要約）
1. モノレポ雛形＋共通レイアウト／ヘッダ／フッタ／グローバルCSS を作成。
2. 8ページ構成で大濠サイトを実装 → note RSS 連携（`fetchNoteNews`、未設定でも空表示でビルド可）。
3. 原田サイトを複製（店舗データのみ差し替え）。
4. ブランドカラーを `BRAND.md` 化し `global.css` に反映（Teal見出し／Gold差し色／Coral CTA）。
5. 大濠の Instagram 実アカウント（@anella_cafe_fukuoka_ohori）をリンク設定。
   画像は自動取得不可のため `public/images/` への手動配置で差し込める受け皿を用意。
6. Instagram **公式埋め込み（embed.js・無料）**コンポーネントを追加（投稿URLを入れると表示）。
7. ドリンクバー方針によりメニューページを廃止し、トップに「ご利用案内」を追加（7ページ構成に）。
8. 内部リンクを base パス対応（`withBase`）に変更し、サブパス公開にも対応（本番ルート公開では無影響）。
9. （仮確認用に GitHub Pages 自動デプロイを試作したが、最終的に Cloudflare 採用のため撤去）

## 検証
- `pnpm -r build` で両サイトのビルド成功（各7ページ）。
- Playwright で全ページ＋モバイル表示をスクリーンショット確認（レスポンシブ・ハンバーガーメニュー動作）。
- note 未設定・画像未配置でもビルドが通ることを確認。
